# KY-040 旋转编码器 Mind+ 扩展库
本库所有功能通过cursor编译实现

Mind+ v1.8 用户库，用于 **KY-040 旋转编码器模块**。

## 加载方式

1. Mind+ v1.8 → **上传模式** → **扩展** → **本地加载** → 选择 `cucucursor-cucuky040-thirdex-V1.4.2.mpext`
2. 若曾加载过旧版，请先在扩展里移除旧库再加载


库身份：`author=cucucursor`，`id=cucuky040`

## 支持模式与主控

### Arduino C

| 主控 / 套件 | board 标识 | 推荐 CLK / DT / SW |
|-------------|------------|---------------------|
| Arduino UNO / Nano / Leonardo / Mega | `arduino` 等 | D2 / D3 / D4 |
| micro:bit / 麦昆 / Calliope | `microbit` 等 | P0 / P1 / P2 |
| 掌控板 / FireBeetle ESP32 / ESP32-E | `esp32` 等 | 27 / 26 / 25 |
| 行空板 K10 | `esp32s3bit` | 8 / 9 / 10 |
| FireBeetle ESP8266 | `esp8266` | D5(14) / D6(12) / D1(5) |
| Maixduino | `maixduino` | 8 / 9 / 10 |
| Raspberry Pi Pico | `pico` | GP2 / GP3 / GP4 |
| 行空板 / handbit | `unihiker` / `handbit` | 27 / 26 / 25 |

### MicroPython

| 主控 | board 标识 | 推荐 CLK / DT / SW |
|------|------------|---------------------|
| Maixduino | `maixduino` | 8 / 9 / 10 |

> MicroPython 模式下请勾选上传库文件（或确保 `MindPlusKY040.py` 已在板端）。

> 引脚可在「初始化」积木下拉菜单中修改。UNO 系建议 CLK/DT 使用中断引脚（如 D2/D3）。

## 接线通则

| KY-040 | 说明 |
|--------|------|
| VCC | 5V 或 3.3V（按主控电平） |
| GND | GND |
| CLK / DT / SW | 见上表推荐引脚 |

## 积木说明

| 积木 | 功能 |
|------|------|
| 初始化 KY-040 | setup 中调用，配置 CLK/DT/SW 引脚 |
| 编码器变换 | 转完**一格**输出 **1** / **-1**，无变化输出 **0**（同轮多次读取一致，约 20ms 后清零） |
| 编码器计数值 | 累计格数（每格 ±1） |
| 设置编码器计数值 | 手动设定当前计数 |
| 编码器计数归零 | 将计数重置为 0 |
| 本次转动步数 | 本轮累计步数 |
| 编码器有转动? | 本轮有转动则为真（同轮多次读取结果一致） |
| 顺时针转动? / 逆时针转动? | 按方向判断（同轮多次读取结果一致） |
| 编码器按键按下? | 当前按键是否按住 |
| 编码器按键被单击? | 检测到一次按下 |
| 编码器按键被松开? | 检测到一次松开 |

## 示例

```
当开机
  初始化 KY-040 CLK[D2] DT[D3] SW[D4]
  编码器计数归零
循环
  如果 编码器变换 ≠ 0
    串口输出 编码器变换
  如果 编码器按键被单击?
    串口输出 按键单击
  等待 10 毫秒
```

> 读取/判断类积木会在内部自动更新编码器，无需手动添加更新积木。

## 使用说明

- AVR 系（UNO/Nano/Mega 等）：CLK/DT 接中断引脚时更不易丢步
- ESP32 系：纯轮询解码，推荐 GPIO27 / 26 / 25
- 行空板 K10（`esp32s3bit`）：推荐 GPIO8 / 9 / 10
- Maixduino MicroPython：推荐引脚 8 / 9 / 10
- 主循环 `等待` 不宜超过 50 毫秒
- 每转一格计数值变化 ±1
- 若方向相反，可交换 CLK/DT 接线，或在程序中对计数值取反
